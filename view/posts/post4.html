Avaliação pessoal sobre o trabalho

Posted by M@lu on June 05, 2017

Durante este primeiro semestre do ano de 2017 estive cursando a matéria Laboratório de Engenharia de Software, nesta disciplina era necessário que fosse desenvolvido um trabalho em grupo que serviria como a avaliação da matéria.

O meu grupo é composto por 4 pessoas: Adler, Rodigo, Yago e Eu. Tinhamos que escolher um cliente para vender o nosso sistema. E escolhemos uma empresa que vende lanches. O sistema proposto para o nosso cliente é uma aplicação mobile para auxiliá-lo no processo de fazer as compras para o estabelecimento.

A maneira como eles fazem esse processo atualmente é o seguinte:

  • Verificar o que falta no estoque.
  • Anotar no papel os itens e quantidades.
  • Ir de mercado em mercado e escrever o valor para cada item.
  • Retornar para o estabelecimento para fazer os orçamentos.
  • Voltar e fazer as compras baseada no menor orçamento.

O que propomos com a nossa aplicação seria realizar esse cálculo de forma automatizada, após ele criar a lista na aplicação e incluir os preços por mercado.

Quando estávamos na etapa de Inception do projeto dividimos o desenvolvimento entre os membros do grupo. As minhas funções seriam relativas ao gerenciamento do projeto e ao design da aplicação. Definido isso, comecei minha parte no desenvolvimento fazendo todas as instalações e configurações necessárias para iniciar o desenvolvimento da aplicação. Veja abaixo com mais detalhes o que foi feito por mim:

  • Criar repositório no GitHub
  • Convidar os outros membros como colaboradores
  • Fazer o PMC (Project Model Canvas)
  • Criar e Configurar o projeto no Taiga
  • Criar e configurar o projeto no Trello
  • Criar um projeto no Slack
  • Relacionar o Github ao Slack
  • Relacionar o TravisCi ao Slack
  • Relacionar o GitHub ao TravisCi e ao SonarQube (Adler assumiu a configuração)
  • Definir o cronograma
  • Definir Sprints (Inicio, Fim, Quantidade, Duração)
  • Criar o arquivo de backlog
  • Reunir a equipe para realizar o planning poker
  • Concluir o arquivo de backlog
  • Atualizar a documentação de entrega relativa a gestão de projetos
  • Projetar o design
  • Iniciar o desenvolvimento do design
